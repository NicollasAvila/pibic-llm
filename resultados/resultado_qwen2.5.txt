### IDENTIFICAÇÃO (Resumo do incidente)
Um uso ilegal de uma API foi detectado pelo Trellix Endpoint Security na máquina SRVEPO, com o IP associado 177.74.1.190, executando um processo PowerShell. A atividade foi detetada duas vezes pela regra "Trellix - Uso de API ilegal" e está relacionada ao uso do comando `AtlComPtrAssign` no contexto do PowerShell.

### ETAPAS DO ATAQUE (Passo a passo técnico)
1. **Identificação da API**: O atacante ou o código malicioso usou uma API específica chamada `AtlComPtrAssign`.
2. **Execução de Comando PowerShell**: Este comando foi executado dentro do PowerShell, provavelmente em um ambiente onde o perfil do usuário (NoProfile) estava desativado.
3. **Detectação pelo Sistema de Detecção de Intrusões (IDS)**: O Trellix Endpoint Security detectou e registrou a atividade suspeita como uma violação das políticas de uso da API.

### AVALIAÇÃO (IoCs, IPs, Hashes)
- **IP Associado**: 177.74.1.190
- **ThreatName**: `ExP:Illegal API Use`
- **TargetProcessName**: `POWERSHELL.EXE`
- **AnalyzerRuleName**: `Powershell Command Restriction - NoProfile`

### MITIGAÇÃO (Ações preventivas)
1. **Verificação do PowerShell Executável**:
   - Verifique se o arquivo `C:\WINDOWS\SYSTEM32\WINDOWSPOWERSHELL\V1.0` e os arquivos relacionados são originais.
   
2. **Auditoria dos Comandos de PowerShell**:
   - Ative a auditoria dos comandos do PowerShell para monitorar quaisquer execuções futuras que possam ser semelhantes ou suspeitas.

3. **Conceitos de Perfil PowerShell (NoProfile)**:
   - Configure o perfil do PowerShell (`$PROFILE`) para evitar execuções anônimas e controle a autenticação necessária.
   
4. **Revisão de Regras de Segurança**:
   - Revise as regras de segurança que estão em vigor, garantindo que elas sejam adequadas e seguras.

5. **Atualizações do Sistema e Software**:
   - Verifique se há atualizações disponíveis para o sistema operacional Windows Server 2012 e qualquer software relacionado.
   
6. **Monitoramento de API’s**:
   - Monitore o uso das APIs existentes, especialmente aquelas com permissões elevadas.

### CONTENÇÃO (Ações imediatas)
1. **Bloqueio do IP Suspeito**: 
   - Bloqueie o IP 177.74.1.190 nas firewall interna e externa.
   
2. **Isolamento da Máquina**:
   - Isole a máquina SRVEPO imediatamente para evitar que qualquer impacto adicional ocorra.

3. **Análise Forense**:
   - Realize uma análise forense nos logs e registros locais e do sistema de gerenciamento (SRVEPO) para identificar se há outras atividades suspeitas.

4. **Notificação de Outros Serviços Afectados**:
   - Informe a equipe de segurança e operações sobre o incidente para uma avaliação mais ampla da exposição potencial.
   
5. **Implementação de Soluções Antivírus Atualizadas**:
   - Verifique se há software antivírus ou solução EDR atualizados que possam identificar e bloquear atividades similares no futuro.

6. **Ajustes nas Políticas de API Uso**:
   - Revise e ajuste as políticas de uso de APIs para evitar futuras violações semelhantes, implementando medidas mais rigorosas se necessário.

### Conclusão
Este incidente demonstra que uma atividade anômala foi detectada no uso de uma API no ambiente de Windows Server 2012. As ações mitigadoras e contenciosas devem ser tomadas imediatamente para proteger o sistema e prevenir futuras vulnerabilidades.

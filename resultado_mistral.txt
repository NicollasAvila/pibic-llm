 ### IDENTIFICAÇÃO (Resumo do incidente)
Incidente de uso suspeito da API em uma máquina Windows Server 2012 com o nome SRVEPO, detectado pelo sistema Trellix Endpoint Security. O comportamento suspeito envolve a execução de comandos Powershell que possivelmente utilizam APIs ilegais.

### ETAPAS DO ATAQUE (Passo a passo técnico)
1. Detectado incidente de uso de API ilegal (ID: 100601) pelo sistema Trellix Endpoint Security.
2. Ocorrência repetida do incidente (firedtimes: 2).
3. Comandos suspeitos relacionados a Powershell foram detectados na máquina SRVEPO.
4. A API suspeita é AtlComPtrAssign e a chamada desta API foi feita no caminho C:\WINDOWS\SYSTEM32\WINDOWSPOWERSHELL\V1.0.

### AVALIAÇÃO (IoCs, IPs, Hashes)
- Máquina: SRVEPO
- Sistema Operacional: Windows Server 2012
- API suspeita: AtlComPtrAssign
- Caminho do arquivo executado: C:\WINDOWS\SYSTEM32\WINDOWSPOWERSHELL\V1.0
- TargetProcessName: POWERSHELL.EXE
- Análise linguística: IDS_NATURAL_LANG_DESC_DETECTION_BOP_2N|APIName=AtlComPtrAssign|TargetPath=C:\WINDOWS\SYSTEM32\WINDOWSPOWERSHELL\V1.0|TargetProcessName=POWERSHELL.EXE|ThreatName=ExP:Illegal API Use

### MITIGAÇÃO (Ações preventivas)
1. Aplicar atualizações de segurança na máquina SRVEPO.
2. Implementar políticas de segurança mais estritas para a execução de comandos Powershell, particularmente quando utilizam APIs suspeitas.
3. Realizar uma auditoria detalhada das APIs instaladas na máquina SRVEPO para remoção ou restringir as que são inseguras.

### CONTENÇÃO (Ações imediatas)
1. Monitorar ativamente a máquina SRVEPO e analisar logs adicionais.
2. Determinar se há outras máquinas infectadas no mesmo ambiente de rede com o mesmo comportamento suspeito.
3. Bloquear a API AtlComPtrAssign ou bloqueie as chamadas à API para evitar a continuação do incidente.
4. Avisar ao time de operações sobre as medidas tomadas e a necessidade de monitoramento adicional da máquina SRVEPO.
